### Student Activity: Hands-On Practice with AWS CLI

This activity is designed to help beginners get hands-on experience with AWS CLI. Follow the steps below to install, configure, and use AWS CLI on your platform. Each section includes practical examples to reinforce your understanding.

---

### **1. Installing AWS CLI**

#### **Windows Installation**

1. **Download and Install AWS CLI v2:**
   - Visit the [AWS CLI v2 for Windows](https://aws.amazon.com/cli/) page.
   - Download the `.msi` installer and run it.
   - Follow the installation instructions.

2. **Verify Installation:**
   - Open Command Prompt and type:
     ```bash
     aws --version
     ```
   - Example Output:
     ```
     aws-cli/2.2.0 Python/3.8.8 Windows/10 exe/AMD64
     ```

#### **macOS Installation Using Homebrew**

1. **Install Homebrew (if not installed):**
   - Open Terminal and run:
     ```bash
     /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
     ```

2. **Install AWS CLI:**
   - Run the following command:
     ```bash
     brew install awscli
     ```

3. **Verify Installation:**
   - Type in Terminal:
     ```bash
     aws --version
     ```

#### **Linux Installation Using Package Managers**

1. **For Ubuntu/Debian:**
   - Update package list and install AWS CLI:
     ```bash
     sudo apt update
     sudo apt install awscli
     ```

2. **For CentOS/RedHat:**
   - Install AWS CLI:
     ```bash
     sudo yum install awscli
     ```

3. **Verify Installation:**
   - Run:
     ```bash
     aws --version
     ```

---

### **2. Configuring AWS CLI with AWS Credentials**

1. **Obtain AWS Credentials:**
   - Log in to AWS Management Console.
   - Navigate to IAM and create a new user or use an existing one.
   - Generate an Access Key and Secret Key.

2. **Configure AWS CLI:**
   - Run the configuration command:
     ```bash
     aws configure
     ```
   - Enter the following when prompted:
     - **AWS Access Key ID**
     - **AWS Secret Access Key**
     - **Default region name** (e.g., `us-east-1`)
     - **Default output format** (e.g., `json`)

3. **Testing Configuration:**
   - List S3 buckets to verify:
     ```bash
     aws s3 ls
     ```

---

### **3. Using AWS CLI Commands**

#### **Example 1: Managing S3 Buckets**

1. **Create a New S3 Bucket:**
   ```bash
   aws s3 mb s3://your-bucket-name
   ```

2. **List All S3 Buckets:**
   ```bash
   aws s3 ls
   ```

3. **Delete an S3 Bucket:**
   ```bash
   aws s3 rb s3://your-bucket-name --force
   ```

#### **Example 2: EC2 Instance Management**

1. **Describe EC2 Instances:**
   ```bash
   aws ec2 describe-instances
   ```

2. **Start an EC2 Instance:**
   ```bash
   aws ec2 start-instances --instance-ids i-1234567890abcdef0
   ```

3. **Stop an EC2 Instance:**
   ```bash
   aws ec2 stop-instances --instance-ids i-1234567890abcdef0
   ```

#### **Example 3: IAM User Management**

1. **List IAM Users:**
   ```bash
   aws iam list-users
   ```

2. **Create a New IAM User:**
   ```bash
   aws iam create-user --user-name new-user
   ```

3. **Delete an IAM User:**
   ```bash
   aws iam delete-user --user-name new-user
   ```

---

### **4. Managing Multiple AWS Profiles**

1. **Create a Named Profile:**
   - Use the `--profile` flag:
     ```bash
     aws configure --profile dev
     ```

2. **Switch Between Profiles:**
   - Specify the profile when running commands:
     ```bash
     aws s3 ls --profile dev
     ```

3. **List All Profiles:**
   - Check the credentials file:
     - On macOS/Linux: `~/.aws/credentials`
     - On Windows: `C:\Users\<YourUsername>\.aws\credentials`

---

### **Conclusion**

By completing this activity, you should now have a practical understanding of how to install, configure, and use AWS CLI on your platform. Practice these commands regularly to become proficient in managing AWS services through the command line.
